## 1. 项目概述

本项目完成了一套基于 STM32F103C8T6 的智能手表原型系统。系统以 FreeRTOS 管理时间、传感器、蓝牙和界面等任务；使用 OLED 显示菜单、秒表、时钟、MPU6050 三轴加速度与步数；使用 EC11 旋转编码器完成菜单选择；通过 HC-05 蓝牙模块与手机串口助手同步步数和查询状态。

最终实物采用面包板搭建，供电方案为充电宝通过 USB 直接给开发板供电。项目未使用锂电池、充电保护模块和外壳，验收重点放在功能稳定性、交互效果和软硬件联调结果。

## 4. 创新点与特色

### 4.1 面向小屏幕的编码器菜单交互

系统没有将编码器仅用于翻页，而是实现了“旋转移动光标、按下选择、按键返回”的菜单操作。四个功能项统一放置在同一页，光标采用小箭头显示；光标移动时仅擦除和重绘两行箭头，不进行整屏刷新，减少 OLED 闪烁并提高操作连贯性。

### 4.2 FreeRTOS 多任务与显示局部刷新结合

时间维护、传感器采样、蓝牙通信和界面交互被拆分为独立任务。界面仅在数据变化时刷新相关区域，使传感器数据、步数和蓝牙通信可以同时运行，同时避免全屏重复清除带来的可见闪烁。

### 4.3 计步数据的本地显示与无线同步

项目将 MPU6050 加速度数据用于阈值式计步，并将结果同时呈现在 OLED 和手机端。手机既能发送命令查询或清零步数，也能在步数变化时自动接收更新。本次验收测试中，计步准确度约为 90%，满足课程设计原型的演示需求。

### 4.4 低门槛、可复现的供电与展示方案

最终采用充电宝 USB 直接供电，规避了锂电池充电、保护和稳压电路在短周期课程设计中可能引入的安全与调试风险。该方案便于现场验收和长期演示，且保持了系统供电稳定。

## 5. 关键实现说明

### 5.1 硬件连接

| 模块 | STM32 接口 | 说明 |
| --- | --- | --- |
| OLED | PB6 / PB7 | 软件 I2C，显示菜单和各功能页面 |
| MPU6050 | PB10 / PB11 | I2C2，读取三轴加速度数据 |
| HC-05 | PA9 / PA10 | USART1，9600 bps 蓝牙通信 |
| EC11 编码器 A/B | PB15 / PB14 | GPIO 上拉输入，读取旋转方向 |
| EC11 编码器按键 | PA8 | GPIO 上拉输入，低电平有效 |
| 返回、清零等按键 | PA1 / PA3 | GPIO 上拉输入 |

### 5.2 软件任务

| 任务 | 主要职责 |
| --- | --- |
| 时间任务 | 维护系统运行时间、软件时钟与秒表时间 |
| 传感器任务 | 初始化并读取 MPU6050，更新加速度和步数 |
| 蓝牙任务 | 处理查询、清零命令，步数变化时自动发送数据 |
| 界面任务 | 处理按键和编码器，绘制菜单与各功能页面 |

## 6. 验收结论与后续改进

系统已完成题目基本要求，并完成旋转编码器菜单、HC-05 蓝牙同步和 MPU6050 计步三项扩展功能。最终实物经充电宝供电后运行稳定，已顺利通过验收。

现阶段原型的主要不足是未制作 PCB 和外壳，计步算法仍为阈值法。后续可在不改变现有软件结构的基础上增加滤波、动态阈值、数据存储和低功耗电池供电模块，并将面包板连接转为 PCB 或洞洞板固定连接。
